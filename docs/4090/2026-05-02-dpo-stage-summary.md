# 2026-05-02 DPO 阶段总结：现象、讨论与收束

这份文档用于收束当前项目在 `DPO` 阶段的观察、讨论与阶段性结论。

目标不是证明 `DPO` 完全无效，而是明确：

1. 当前这批 DPO 数据和训练在实际实验中呈现了什么现象
2. 为什么这些现象说明当前 DPO 信号还不够干净
3. 为什么对一个以学习为主的项目来说，当前阶段可以合理停在这里

## 1. 训练现象

当前 `4B` 与 `8B` 的 DPO 训练都能顺利跑通，但从训练曲线和中间指标看，整体呈现出几个共同特点：

### 1.1 曲线震荡明显

从 W&B 图上可以看到：

- `train/loss` 波动较大
- `train/rewards/margins` 没有形成特别稳的单调上升趋势
- `train/rewards/accuracies` 在不同 step 间有明显抖动

尤其在把 `medium` 置信度样本也加入后：

- `4B` 的曲线抖动更明显
- loss 后段会重新抬头
- margin 的稳定性变差

这说明：

- 训练不是“完全学不会”
- 但偏好信号并没有稳定地朝单一方向推动模型

### 1.2 4B 与 8B 的响应方式不同

当前观察到：

- `4B` 的 DPO train loss 更低，reward 指标提升也更明显
- `8B` 也能学，但收益更像温和修正

这说明：

- 小一些的模型更容易被当前这批偏好对“推着走”
- 更强的模型则更像是在强基线之上被做局部精修

### 1.3 自动评测与 judge 给出的结论不完全一致

在 `HF` 自动规则评测里：

- `4B DPO` 相比 `4B SFT` 的提升并不明显
- `8B DPO` 主要体现为结构稳定性略有补强

但在 `SFT vs DPO` 的 pairwise judge 里：

- `4B DPO` 对 `4B SFT` 有更明显的胜率提升
- `8B DPO` 对 `8B SFT` 也有正向收益，只是更多体现为温和精修

这说明：

- DPO 的收益并不总能直接反映在简单自动指标上
- 当前这批 DPO 数据更容易影响“工程化质量”，而不一定明显改变类别/严重度这种粗粒度自动指标

## 2. 为什么会这样

当前阶段我们对问题的理解已经比较清楚：

### 2.1 偏好目标不够单一

当前任务的 DPO pair 不是围绕一个非常单一的目标构造的，例如：

- 安全性
- 礼貌性
- 风格一致性

而是一个明显更复合的目标：

- 更 grounded
- 更保守
- 更会承认不确定性
- 更会补 missing_info
- 更会给可执行的 next_steps
- 更像靠谱工程师
- 同时还隐含了一部分结构稳态偏好

这意味着：

- pair 的 winner 不一定总是因为同一种原因
- 模型学到的是“混合偏好”
- 而不是特别纯的一条偏好轴

### 2.2 `medium` 样本会引入更多噪声

当前实验已经看到：

- `high_only` 更稳
- `high + medium` 更容易抖

这说明：

- `medium` 样本并不是完全没价值
- 但它们的边界更模糊，噪声也更大
- 对较小模型尤其容易造成训练曲线震荡

### 2.3 DPO 比 SFT 更吃数据纯度

SFT 往往允许：

- 数据风格有一些噪声
- 目标不那么单一

但 DPO 不一样。

它优化的是：

- `chosen` 相对 `rejected` 的偏好差异

所以如果：

- pair 差异过粗
- pair 差异过杂
- judge 虽稳定但目标轴不够纯

训练就容易出现：

- loss 有下降
- 但收益有限
- 曲线波动较大
- 评测结果不好解释

## 3. 这是否说明 DPO 没有价值

不是。

更准确的说法是：

- 当前这批数据和训练证明了 `DPO` 在这个任务上**可行**
- 但也同时证明了：
  - 它的收益有限
  - 它比 SFT 更吃数据工程
  - 它需要更清晰、更单一的偏好定义

所以当前最稳的判断不是：

- `DPO` 完全无效

而是：

- **当前这批 DPO 数据更像一版 exploratory dataset**
- 它已经足够完成学习与验证目的
- 但还没有成熟到足以成为项目下一阶段的主轴投入对象

## 4. 为什么现在可以停在这里

对于一个以学习为主的项目来说，当前阶段已经得到的东西很多：

### 4.1 已经跑通了完整闭环

项目已经完成了：

- 数据构建
- SFT
- 自动规则评测
- LLM-as-a-judge
- DPO 数据生成
- DPO 数据清洗与分层
- DPO 训练
- SFT vs DPO 对比
- 更大模型 scale-up
- 推理基础设施边界探索

这已经是一个非常完整的实验闭环。

### 4.2 再往下的主要成本开始转向数据工程

继续深挖 DPO 的下一步，主要不再是“再跑几轮训练”，而会变成：

- 定义更清晰的偏好轴
- 重做更纯的 pair
- 区分不同子目标的数据池
- 再次大规模生成和筛选数据

这部分当然有价值，但投入的工程复杂度和时间成本会显著上升。

### 4.3 当前已经有足够可讲的结论

当前阶段已经可以比较稳地说：

- `4B -> 8B -> 14B` 的 scale-up 在训练信号上是有意义的
- `14B` 已接近当前单卡训练甜点位上沿
- DPO 在当前任务上可行，但收益有限且依赖数据质量
- 小模型更像被 DPO 明显拉升，大模型更像被 DPO 温和精修

这已经足够支持：

- 一段完整的项目总结
- 一段实习 / 面试中的技术项目叙述

## 5. 当前阶段结论

如果要把当前 DPO 阶段压缩成一句更准确的话：

**当前 DPO 训练不是“完全不起作用”，而是“收益存在但有限，且高度依赖偏好数据的纯度与目标清晰度”；对于当前这个以学习为主的项目，这已经足够证明问题所在，也足够支撑在这里阶段性收束。**

## 6. 一句话结论

这轮 DPO 阶段最重要的收获不是“把模型推到了一个全新水平”，而是：

**把当前任务中 DPO 的可行性、收益形态和主要瓶颈都摸清楚了。对于一个以学习为主的项目来说，这已经是一个很好的停点。**
